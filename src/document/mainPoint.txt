2....✅ Method 1: Tailwind’s dark: classes (recommended)
Set darkMode: 'class' in tailwind.config.js
Then toggle a class like dark on <html> or <body>
<div className="bg-white dark:bg-gray-900 text-black dark:text-white">Dark mode works!</div>

✅ Method 2: Manually toggle classNames using JavaScript
<div className={`${darkMode ? 'bg-gray-900 text-white' : 'bg-white text-black'}`}>
  Dark mode manually applied
</div>




3.....✅ When to Use dark: vs. Not Use It
Use dark: When... ✅	                                   Don't Use dark: When... ❌
You want different styles in dark vs. light mode	      You want the same style in both modes
Example: bg-white in light, bg-gray-900 in dark    	      Example: text-center, py-6, rounded-lg, etc.
Color properties like bg-, text-, border-	              Layout properties like flex, grid, gap-4, etc.
Tailwind supports it on color-based utilities	          Utility isn’t color-dependent (like text-sm, w-full)

6----  Why NavLink is Better for Navigation Menus

Highlights the current page:
NavLink gives you access to {isActive} which tells you whether the link's path matches the current URL.

| Feature                     | `Link`                       | `NavLink`                    |
| --------------------------- | ---------------------------- | ---------------------------- |
| Basic navigation            | ✅ Yes                        | ✅ Yes                        |
| Style the active route link | ❌ No built-in support        | ✅ Yes (has `isActive` state) |
| Detect current page         | ❌ Not possible               | ✅ Automatically detects      |
| Ideal for menus/navbars     | 🚫 Only for basic navigation | ✅ Best for navbars/menus     |
